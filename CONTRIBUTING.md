# Contributing to Z-CMS Documentation

Thank you for helping improve Z-CMS documentation.

## Make a change

1. Create a focused branch.
2. Edit Markdown or MDX under `src/content/docs/`.
3. Keep translated pages at the same relative path in every locale.
4. Run `pnpm build`.
5. Open a pull request that describes the audience and outcome of the change.

Product behavior and its documentation should be updated in the same release. Never include real credentials, private customer data or secrets in examples.
