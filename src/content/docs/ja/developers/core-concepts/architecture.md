Z-CMS では、API、公開サイトのレンダラー、Admin、プラグインランタイム、worker をそれぞれ独立したプロセスとして実行します。

## コンポーネント

| コンポーネント | 責務 |
| --- | --- |
| `cms-api` | 認証、RBAC、コンテンツ、メディア、マーケットプレイス、Render API |
| `site-runtime` | 有効なテーマで公開サイトをレンダリング |
| `admin-web` | 管理インターフェース |
| `plugin-runtime` | 認証情報を保持せず、V8 isolate 内でプラグインを実行 |
| `worker` | 画像バリエーション、メール、サイトマップ、遅延ジョブ |

## 信頼境界

- テナント分離は PostgreSQL Row-Level Security によって強制されます。
- プラグインのコードは API プロセス内では実行されません。
- パッケージは公開者が署名し、Marketplace が審査・署名した後に配布されます。インストール時には、Z-CMS が両方の署名とチェックサムを検証します。

これらはプラットフォームが保証する必須条件であり、拡張機能側で省略できる任意のルールではありません。
