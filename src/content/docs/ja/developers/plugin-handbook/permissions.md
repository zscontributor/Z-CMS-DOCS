プラグインはマニフェストで権限を宣言します。権限が有効になるのは、Administrator が承認した後です。ランタイムには有効期間が短く利用範囲を限定したトークンが渡され、CMS ゲートウェイはリソースへのアクセスを許可する前にその範囲を再検証します。

## 原則

- 現在の機能に必要な権限だけを要求します。
- Administrator が一部の権限だけを承認した場合にも対応します。
- トークンをストレージやログに保存しません。
- 権限が増える変更はリリースノートに明記します。

権限の確認を UI だけで完結させてはいけません。CMS は信頼境界で必ず権限を検証します。
