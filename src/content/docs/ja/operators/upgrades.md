アップグレード前にリリースノートを読み、テーマとプラグインの互換性を確認してください。あわせて、復元テストに成功したバックアップを用意します。

## 推奨手順

1. 機密情報を削除またはマスキングした本番データのコピーで、新しいバージョンをテストします。
2. ステージング環境でマイグレーションとスモークテストを実行します。
3. マイグレーションで必要な場合は、スキーマの変更を一時停止します。
4. リリースノートに記載された順序で各コンポーネントをデプロイし、互換性を維持します。
5. API の状態、キュー、レンダリング、監査イベントを確認します。
6. ロールバック可能期間が終了してから、古いバージョンのリソースを削除します。
