テーマは、公開サイトのテンプレート、ブロック、メニューの配置場所、アセット、SEO メタデータを定義します。テーマはシステム単位でインストールし、サイトごとに有効化・設定します。

## テーマを変更する前に

1. サイトスイッチャーで現在のサイトを確認します。
2. テーマの説明、バージョン、互換性を読みます。
3. 使用中のコンテンツタイプに対応するテンプレートがあることを確認します。
4. 新しいテーマで問題が発生したときに元に戻せるよう、現在のテーマ設定を記録しておきます。すでに公開中のサイトでは、本番サイトに適用する前にステージングサイトで新しいテーマをテストしてください。

## インストールと有効化

**Appearance** には、インストール済みのテーマとカタログから追加できるテーマが表示されます。`theme:install` 権限があればパッケージをインストールでき、`theme:activate` 権限があれば有効なテーマを切り替えられます。テーマを有効化するとサイトのレンダーキャッシュは更新されますが、コンテンツは削除されません。

## テーマを設定する

Z-CMS は、有効なテーマの JSON Schema から設定フォームを生成します。表示される項目はテーマによって異なり、色、書体、テーマ固有のロゴ、SNS リンク、レイアウト、favicon、SEO 設定などがあります。

- サイト共通ブランドを継承する場合は、ロゴや色の上書きを空欄にします。
- 一時的な URL ではなく Media Library からアセットを選びます。
- 保存後、デスクトップとモバイルで詳細ページや 404 ページも確認します。

テーマによっては **Seed demo content** 機能を提供しています。コンテンツタイプ、ページ、記事、メニュー、テーマ設定が追加される可能性があるため、新規サイトまたはステージング環境でのみ使用してください。

## テーマが管理する SEO

テーマは、テーマ設定とサイトのブランド情報を使ってタイトルテンプレート、アイコン、robots ポリシー、Open Graph、JSON-LD を生成できます。テーマを変更した後は、ページソースと SNS 共有時のプレビューを確認してください。

:::note
インストール、有効化、設定のボタンや form が無効な場合は、アカウントが読み取り専用か、対象とは異なるサイトを選択している可能性があります。
:::
