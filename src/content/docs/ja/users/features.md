Z-CMS はマルチテナントシステムです。1 つのプラットフォームで複数の Web サイトを管理しながら、データと設定はサイトごとに分離されます。Admin に表示される機能は、アカウントのロールと選択中のサイトによって異なります。

| 領域 | できること | ガイド |
| --- | --- | --- |
| Sites | サイト作成、ホスト名、ブランド、公開状態の管理 | [複数サイトの管理](/ja/users/sites/) |
| Content | ページやブログの作成、ブロック、翻訳、プレビュー、公開 | [コンテンツと公開](/ja/users/content/) |
| Media | アップロード、検索、フォルダー整理、代替テキスト編集 | [メディア](/ja/users/media/) |
| Marketplace | 署名済みパッケージの検索とインストール | [テーマとプラグイン](/ja/users/extensions/) |
| Appearance | テーマの有効化、スキーマによる設定、デモデータ作成 | [外観とテーマ設定](/ja/users/appearance/) |
| Users / Profile | 招待、ロール、パスワード、2FA | [ユーザーとアカウント保護](/ja/users/users-security/) |
| Settings → Mail | SMTP 設定とテスト送信 | [メールとバックグラウンドジョブ](/ja/users/operations/) |
| Jobs | 失敗したジョブの確認と再実行 | [メールとバックグラウンドジョブ](/ja/users/operations/) |
| zAI | 自然言語によるページやブログの管理 | [zAI Content Operator](/ja/users/zai/) |

## 変更対象を確認する

サイトスイッチャーで選択しているサイトが、コンテンツ、メディア、外観、プラグイン、メール設定の変更対象になります。一方、**Sites** 画面には、アカウントがアクセスできるすべてのサイトが表示されます。必要なメニューや操作ボタンが表示されない場合は、不具合と判断する前に、選択中のサイトと自分のロールを確認してください。

## バックグラウンドコンポーネント

Worker は画像バリエーション、メール、サイトマップ、プラグインフックを処理します。Marketplace は失効したパッケージの情報を同期します。監査ログには、zAI 経由の操作を含むセキュリティイベントと重要な操作が記録されます。
